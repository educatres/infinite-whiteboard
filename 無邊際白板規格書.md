# 純 HTML + JavaScript 無邊際白板開發規格書

## 一、專案目標

開發一個可直接部署於 GitHub Pages 的純前端靜態網頁白板工具。

本白板不需要後端、不需要資料庫、不需要登入，也不需要 npm build。使用者可以用滑鼠、觸控筆或手指在白板上繪圖，並支援無邊際白板、雙指縮放、平移、匯出 PNG、匯出 JSON、匯入 JSON 重建白板等功能。

---

## 二、技術限制

本專案需符合以下條件：

1. 僅使用 HTML、CSS、JavaScript。
2. 不使用後端伺服器。
3. 不使用資料庫。
4. 不使用 npm、Webpack、Vite 或其他 build 工具。
5. 可直接用瀏覽器開啟 `index.html`。
6. 可直接部署到 GitHub Pages。
7. 優先使用原生 Canvas 2D API。
8. 優先使用 Pointer Events 支援滑鼠、觸控、觸控筆。

---

## 五、無邊際白板設計

### 1. 座標系統

白板需使用「世界座標」儲存所有筆跡。

畫面上使用者看到的是「螢幕座標」，實際儲存時需轉換成世界座標。

需要維護一個 viewport 狀態：

```js
const viewport = {
  x: 0,
  y: 0,
  zoom: 1
};
```

其中：

```text
viewport.x：目前畫面中心對應的世界座標 x
viewport.y：目前畫面中心對應的世界座標 y
viewport.zoom：目前縮放比例
```

### 2. 座標轉換

需要實作兩個函式：

```js
function screenToWorld(screenX, screenY) {
  return {
    x: (screenX - canvas.width / 2) / viewport.zoom + viewport.x,
    y: (screenY - canvas.height / 2) / viewport.zoom + viewport.y
  };
}

function worldToScreen(worldX, worldY) {
  return {
    x: (worldX - viewport.x) * viewport.zoom + canvas.width / 2,
    y: (worldY - viewport.y) * viewport.zoom + canvas.height / 2
  };
}
```

所有筆跡都應儲存世界座標，不應儲存螢幕座標。

## 六、平移與縮放功能

### 1. 滑鼠平移

需支援以下方式之一：

1. 按住空白鍵拖曳畫布平移。
2. 使用「平移工具」拖曳畫布平移。
3. 滑鼠中鍵拖曳畫布平移。

建議至少支援「空白鍵 + 滑鼠拖曳」。

### 2. 觸控平移

在觸控裝置上：

1. 單指使用目前選取的繪圖工具。
2. 雙指進入平移與縮放模式。
3. 雙指拖曳時可以平移白板。
4. 雙指距離改變時可以縮放白板。

### 3. 雙指縮放

需使用 Pointer Events 追蹤多指觸控。

建議維護一個 Map：

```js
const activePointers = new Map();
```

當 `pointerdown` 時加入 pointer。

當 `pointermove` 時更新 pointer。

當 `pointerup`、`pointercancel`、`pointerleave` 時移除 pointer。

當 activePointers 數量為 2 時，進入雙指操作模式。

雙指縮放需計算：

```text
上一幀雙指距離 oldDistance
目前雙指距離 newDistance
縮放倍率 scale = newDistance / oldDistance
```

更新：

```js
viewport.zoom *= scale;
```

需限制縮放範圍：

```js
const MIN_ZOOM = 0.1;
const MAX_ZOOM = 5;
```

### 4. 以雙指中心為縮放中心

雙指縮放時，畫面應以兩指中心點作為縮放中心，避免縮放時白板內容跳動。

做法：

1. 計算雙指中心點在螢幕上的座標。
2. 縮放前，把該中心點轉成世界座標。
3. 更新 zoom。
4. 縮放後，再計算同一個世界座標應如何維持在原螢幕位置。
5. 反推 viewport.x 與 viewport.y。

邏輯範例：

```js
function zoomAt(screenX, screenY, newZoom) {
  const before = screenToWorld(screenX, screenY);

  viewport.zoom = Math.min(MAX_ZOOM, Math.max(MIN_ZOOM, newZoom));

  const after = screenToWorld(screenX, screenY);

  viewport.x += before.x - after.x;
  viewport.y += before.y - after.y;

  redraw();
}
```

雙指縮放時：

```js
const newZoom = viewport.zoom * scale;
zoomAt(centerX, centerY, newZoom);
```

### 5. 滑鼠滾輪縮放

桌機版需支援滑鼠滾輪縮放。

使用者滾動滑鼠滾輪時，以滑鼠所在位置為縮放中心。

```js
canvas.addEventListener("wheel", (event) => {
  event.preventDefault();

  const zoomFactor = event.deltaY < 0 ? 1.1 : 0.9;
  const newZoom = viewport.zoom * zoomFactor;

  zoomAt(event.offsetX, event.offsetY, newZoom);
}, { passive: false });
```

## 八、繪圖互動流程

### 1. 自由筆刷

流程：

1. pointerdown：建立新的 stroke。
2. pointermove：持續加入世界座標點。
3. pointerup：結束 stroke，存入 strokes。
4. 每次 move 後 redraw。

### 2. 橡皮擦

流程與自由筆刷相同，但 type 為 eraser。

### 3. 直線、長方形、圓形、橢圓形

流程：

1. pointerdown：記錄 start 世界座標。
2. pointermove：更新 preview end 世界座標。
3. redraw 時先畫已完成 strokes，再畫 preview。
4. pointerup：將圖形存入 strokes。
5. 清除 preview。

## 九、重繪機制

需實作：

```js
function redraw() {
  clearCanvas();
  drawGrid();
  drawAllStrokes();
  drawPreviewShape();
}
```

### 1. 清除畫布

```js
ctx.clearRect(0, 0, canvas.width, canvas.height);
```

### 2. 背景

背景為白色。

```js
ctx.fillStyle = "#ffffff";
ctx.fillRect(0, 0, canvas.width, canvas.height);
```

### 3. 格線

建議加入淡色格線，幫助使用者理解無邊際白板正在平移與縮放。

格線需跟著世界座標移動與縮放。

可設定：

```js
const GRID_SIZE = 50;
```

格線顏色建議使用淡灰色。

### 4. 繪製筆跡

每次 redraw 時，依序繪製 strokes 陣列中的所有項目。

每一筆資料都要透過 `worldToScreen()` 轉成畫面座標後再畫到 Canvas。

## 十、PNG 匯出

需支援 PNG 匯出。

### 第一階段最低需求

匯出目前使用者畫面中看到的白板內容。

```js
const dataUrl = canvas.toDataURL("image/png");
```

下載檔名：

```text
whiteboard.png
```

### 第二階段建議需求

支援匯出所有筆跡的完整範圍。

做法：

1. 根據 strokes 計算所有筆跡的 bounding box。
2. 建立 offscreen canvas。
3. 將所有筆跡重新繪製到 offscreen canvas。
4. 匯出 PNG。

第二階段可先不做，但程式架構需保留擴充空間。

## 十一、JSON 匯出

需支援將目前白板資料匯出成 JSON 檔案。

JSON 格式：

```json
{
  "version": "1.0",
  "createdAt": "2026-07-09T00:00:00.000Z",
  "viewport": {
    "x": 0,
    "y": 0,
    "zoom": 1
  },
  "strokes": []
}
```

下載檔名：

```text
whiteboard.json
```

匯出邏輯：

```js
function exportJSON() {
  const data = {
    version: "1.0",
    createdAt: new Date().toISOString(),
    viewport,
    strokes
  };

  const blob = new Blob([JSON.stringify(data, null, 2)], {
    type: "application/json"
  });

  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = "whiteboard.json";
  a.click();
  URL.revokeObjectURL(url);
}
```

## 十六、Canvas 尺寸與 Retina 支援

Canvas 需支援高解析度螢幕，避免畫面模糊。

resize 時：

```js
function resizeCanvas() {
  const dpr = window.devicePixelRatio || 1;

  canvas.width = window.innerWidth * dpr;
  canvas.height = window.innerHeight * dpr;

  canvas.style.width = window.innerWidth + "px";
  canvas.style.height = window.innerHeight + "px";

  ctx.setTransform(1, 0, 0, 1, 0, 0);
  ctx.scale(dpr, dpr);

  redraw();
}
```

注意：若使用 CSS 像素進行座標計算，需確保 `screenToWorld()` 使用的是 CSS 尺寸，而不是實際 canvas pixel 尺寸。

建議維護：

```js
const canvasCssWidth = canvas.clientWidth;
const canvasCssHeight = canvas.clientHeight;
```

或在座標轉換函式內使用：

```js
canvas.clientWidth
canvas.clientHeight
```

5. JSON 匯出需包含 strokes 與 viewport。
6. JSON 匯入後應立即 redraw。
7. 觸控操作必須設定 `touch-action: none`。
8. 滾輪縮放與雙指縮放都應限制在 `0.1` 到 `5` 倍。
9. 程式碼需盡量清楚，方便後續加入多人同步或 Google Sheet 儲存功能。
10. 不要引入大型框架，保持純前端、輕量、容易維護。

# 開發任務總結

請實作一個純 HTML + CSS + JavaScript 的無邊際白板。

白板需要支援滑鼠、觸控筆與手指繪圖，提供筆刷、橡皮擦、直線、長方形、圓形、橢圓形工具。使用者可以切換顏色與粗細，可以用滑鼠滾輪縮放，也可以用雙指縮放與雙指平移。所有筆跡需使用世界座標儲存，並能匯出成 JSON，再匯入 JSON 重建白板。白板也需支援匯出目前畫面為 PNG 圖片。整個專案必須是純靜態網頁，可直接部署到 GitHub Pages。
